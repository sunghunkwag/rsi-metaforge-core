# Phase J1 — ISA Gap Analysis for the Certified Walls (T29–T32)

Status: measurement-only deliverable (DIRECTIVE v5, Phase J1). No mechanism
code accompanies this document. Every claim below is either (a) pinned by an
existing committed test, (b) read off a committed artifact, or (c) reproducible
by the stated command; interpretation beyond the proofs is labeled as such.

## 1. The committed certificates

Four independent, machine-checked instruments define where the current
instruction set ends. All four are enforced by tests that run in every suite
execution (147-test native registry and pytest collection):

1. **Exact closure certificates at expanded length <= 6.**
   `closure_certificate` (rsi_levels_metaforge_unified.py:5342) enumerates
   every base-ISA program of expanded length <= 6 by untruncated
   observational-equivalence classes over all 20 train inputs
   (`CERT_MAX_LEN = 6`, `CERT_CLASS_CAP = 600_000`) and reports, per task,
   whether the task's train vector is reachable.
   `test_closure_certificates_walls_blocked_le6` (line 5753) pins: the wall
   set is exactly the four family-"search" tasks, each certificate is
   untruncated, and each reports `certified_blocked_le == 6`.

2. **Deep scan: exact at length 7, complete reachability sweep at length 8.**
   `targeted_reach_scan` (line 5932) extends the certificate beyond what a
   full in-memory table can hold (a full length-7 table exceeds 4M classes and
   dies of memory; the scanner holds signatures only to length 6 — 332,509
   classes, pinned — and reconstructs deeper frontiers on demand).
   `test_deep_scan_artifact_certifies_walls` (line 6406) pins the artifact
   (`closure_scan.json`, regenerated by `--mode horizon-scan`):
   `truncated == False`, `classes_per_len[5] == 332509`,
   `max_certified_len >= 7`, and `found == {}` — **no base-ISA program of
   expanded length <= 7 (exactly), nor any at the length-8 sweep, computes any
   of the four wall train vectors.**

3. **Executed constructor lemma (absolute, not depth-bounded).**
   `no_int_to_list_constructor_lemma` (line 5356) executes every base op
   against every pure-int stack of depth <= 3 with values 0..3 (>= 2000
   transitions, pinned by `test_no_int_to_list_constructor_lemma`, line 5778)
   and finds that **no base op except `INPUT` ever pushes a list**. Hence
   every list value in any base-ISA program is value-derived from the input
   list, and a program-supplied constant list (broadcast) is inexpressible at
   *any* program length. This is executed, not assumed.

4. **BCAST-augmented certificate.** With the one dormant catalog primitive
   `BCAST` granted (`(int, list) -> list` constant broadcast, ext id 60), the
   walls remain unreachable at expanded length <= 5
   (`test_bcast_certificate_consistent_with_witness`, line 6424, live) and
   <= 7 (horizon report artifact), while T29 becomes *expressible* at expanded
   length 19 via the committed designer witness `T29_BCAST_WITNESS_TOKENS`
   (line 6185) — a witness that passes both sealed gates and is never counted
   as solved (`counts_as_solved = False`, pinned).

Supporting measured context (not proofs): docs/ATTRIBUTION_F.md classifies all

## 2. What the ISA has, for contrast

The base ISA (34 ops, lines 148–161) contains order-**selection** at every
tier — `MAX2`/`MIN2` (scalar), `ZMAX` (elementwise), `SCAN_MAX` (prefix),
`RED_MAX` (reduce), `SORTL` (global) — and a scalar order-**test** is already
inside the closure by composition: for ints `a`, `b` on the stack,
`SUB PUSH0 MAX2 PUSH1 MIN2` leaves the indicator `min(max(a-b, 0), 1)`, i.e.
`1 if a > b else 0`. The list tier has no analogue: the elementwise capping
constants (`0`, `1` as lists) are exactly the program-supplied constant lists
whose construction the executed lemma proves impossible, and no zip op
produces an order indicator directly (`ZADD`, `ZSUB`, `ZMUL`, `ZMAX` are the
whole zip family). Producing a *positional* answer additionally requires
turning order relations into arithmetic-consumable data; the ISA's positional
ops (`HEAD`, `LAST`, `IDX`, `EVENIDX`, `ODDIDX`, `LEN`) consume or measure
positions but no op emits an order test as data.

## 3. Gap statement, one sentence per task

- **T29 `count_above_threshold`** (`sum(1 for x in xs if x > 3)`): the proofs
  show the ISA cannot express an elementwise order test against a
  program-supplied constant — the executed constructor lemma closes the only
  constant-comparand route (no int-to-list constructor exists at any depth)
  and the exhaustive scan certifies no base-ISA program of expanded length
  <= 7 (<= 8 swept) computes the train vector by any other route.

- **T30 `argmax_index`** (first index attaining the maximum): the proofs show
  the ISA cannot express the conversion of order relations into a positional
  value — every order op selects or aggregates *values*, never emits a
  comparison outcome as data a downstream op could count into an index, and
  the exhaustive scan certifies no <= 7 (<= 8 swept) composition of
  value-selection, indexing, and arithmetic expresses the train vector.

- **T31 `first_greater_than_previous`** (first index whose element exceeds its
  predecessor, else -1): the proofs show the ISA cannot express reified
  adjacent-pair order tests — the zip family can form adjacent differences
  (`ZSUB` on `TAIL`) but no op maps them to indicator data whose position
  could be extracted, and the exhaustive scan certifies unreachability at
  expanded length <= 7 (<= 8 swept).

- **T32 `longest_increasing_run`** (length of the longest strictly increasing
  contiguous run): the proofs show the ISA cannot express order-test
  reification, without which segment-local accumulation (a reset-on-boundary
  run measure) has no data to accumulate, and the exhaustive scan certifies
  unreachability at expanded length <= 7 (<= 8 swept).

**Shared structure (interpretation, stated as such):** all four walls factor
through one missing capability — *an elementwise order test whose outcome is
data* (an indicator list). T29 additionally requires the constant comparand
that the lemma proves unconstructible (the absolute gap `(int, list) -> list`
already named by the dormant catalog's BCAST rationale); T30/T31 reduce, once
indicators exist, to positional arithmetic already inside the closure
(`SCAN_MAX`, `RED_ADD`, `LEN`); T32 reduces to segment accumulation already
inside the closure (`SCAN_ADD`, `SCAN_MAX`, `ZMUL`, `ZSUB` algebra).

## 4. Epistemic boundary

The certificates are exact and exhaustive up to expanded length 7 (length 8
complete for reachability) and the constructor lemma is absolute; deeper
base-ISA routes for T30–T32 are excluded by no proof, only by the measured
search record at declared budgets (ATTRIBUTION_F, EXPANSION_RESULT, UNION_H,
SEQUENCING_RESULT). The Section-16 self-forge and Section-11 CFS batteries
demonstrate wall solutions *outside* the base ISA (task-derived fold
primitives; feature-space decoding); per GR5/origin discipline those never
enter the designer ledger and do not bear on this gap analysis.
